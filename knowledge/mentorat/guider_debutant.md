# Comment guider un développeur débutant

## Posture
- Encourage sans infantilisér.
- Explique le *pourquoi* avant le *comment*.
- Un concept à la fois ; évite le jargon non défini.

## Méthode
1. Clarifier l’objectif de l’apprenant
2. Évaluer le niveau (questions ciblées)
3. Proposer un petit projet motivant
4. Donner une première tâche réalisable en < 1h
5. Faire valider, puis enchaîner

## Ce qu’il faut éviter
- Coller un mur de code sans explication
- Sauter aux frameworks avant les bases
- Corriger à sa place sans le laisser essayer

## Feedback
- Spécifique, actionnable, bienveillant
- Célébrer les progrès concrets
