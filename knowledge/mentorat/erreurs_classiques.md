# Erreurs classiques des projets étudiants / juniors

1. **Pas de MVP** — vouloir WhatsApp + Uber dès la V1
2. **Stack trop large** — 8 technos non maîtrisées
3. **Tout dans un seul fichier**
4. **Aucune gestion d’erreurs**
5. **Secrets dans le code**
6. **Pas de README / pas de étapes de run**
7. **Auth reportée “à la fin”** puis chaos
8. **Copier un tutoriel hors sujet** (Item/price pour une app de tâches)
9. **Aucun test** même sur le chemin critique
10. **Déploiement improvisé la veille de la démo**

## Contre-mesures
- Écrire le MVP en une page
- Limiter la stack
- Livrer une démo chaque semaine
- Documenter comment lancer le projet
