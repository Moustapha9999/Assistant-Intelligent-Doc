# Gestion de projet logiciel (pragmatique)

## Découpage
1. Clarifier l’objectif utilisateur (valeur)
2. Définir un MVP (plus petit produit utile)
3. Découper en étapes livrables (vertical slices)
4. Estimer grossièrement (S/M/L) plutôt qu’en heures précises au début

## Priorisation
- Impact utilisateur × risque × effort
- Faire d’abord ce qui débloque le reste (auth, modèle de données, CI)
- Reporter le “nice to have”

## Suivi
- Une source de vérité (board, issues)
- Definition of Done : code + test + doc minimale
- Revue régulière des risques

## Erreurs fréquentes
- Vouloir tout construire d’un coup
- Pas de MVP → scope creep
- Ignorer les risques techniques (auth, multi-tenant, perfs)
- Aucune démo intermédiaire
