# Protocole d'étude utilisateur — étudiants ISI

## Objectif

Évaluer si l'assistant facilite la recherche de documentation et la planification d'un petit projet.

## Déroulement (30 minutes)

1. Relever le temps de recherche habituel avant utilisation (auto-déclaré).
2. Présenter l'outil et laisser 3 minutes de prise en main.
3. Réaliser les cinq tâches ci-dessous et chronométrer chaque tâche.
4. Relever le temps total après utilisation et remplir le questionnaire.

## Tâches

1. Trouver comment créer une route JSON avec Flask.
2. Trouver comment valider une requête FastAPI avec Pydantic.
3. Identifier une bonne pratique pour protéger un JWT.
4. Obtenir les étapes d'une roadmap pour une API REST.
5. Trouver une solution au problème N+1 dans Django.

## Questionnaire Likert (1 = pas du tout d'accord, 5 = tout à fait d'accord)

- Les sources trouvées étaient pertinentes.
- Les réponses étaient faciles à comprendre.
- Les filtres m'ont aidé à réduire le bruit.
- L'outil m'a fait gagner du temps.
- Je réutiliserais l'assistant pour un projet ISI.

Recueillir aussi un commentaire libre, le niveau technique de l'étudiant et les difficultés rencontrées.
